Support Eng · Ticket Triage
01 · Overview

Cut manual sorting to only the tickets automation is unsure about

Every support ticket gets sorted into a category before an agent can work it. This plan adds a rule engine and a single model call so only the unclear cases still need a human to decide.

PurposeClassify every incoming ticket automatically, and only hand off the ones the system isn't confident about.
BoundaryThis system only assigns a category and a confidence score. It does not reply to the customer or resolve the ticket.
SuccessThe pipeline matches human classification at 90% accuracy or higher, measured in shadow mode, before it ever routes a live ticket.
02 · What Changes

100 hours of manual sorting a day, shrunk to the unsure fraction

Support agents read and categorize every ticket today. Rules and a model do that first now; agents step in only when confidence is low.

03 · Classification Pipeline

Keyword rules first, one model call as fallback

Around 50 keyword rules catch the clear-cut tickets instantly. Anything they miss gets exactly one model call, which returns a category and a confidence score.

  1. Rule match. The engine checks the ticket text against ~50 keyword rules. A hit assigns the category immediately, no model call needed.
  2. Model fallback. If no rule matches, exactly one model call classifies the ticket and returns a confidence score.
  3. Confidence gate. A score of 0.7 or higher auto-assigns the category. Below 0.7, the ticket goes to the human review queue.

What the keyword rules look like

RuleKeyword or phraseCategory
01refund, chargeback, money backBilling → Refund
02can't log in, password reset, locked outAccount → Login
03where is my order, shipping delay, trackingLogistics → Delivery
04cancel subscription, close my accountBilling → Cancellation

This is 4 of the roughly 50 rules. The full table is maintained separately and updated as new ticket patterns show up.

04 · Workflow

Confidence decides who classifies the ticket

Every ticket follows the same five steps from submission to a final category. Only the last step branches between automation and a person.

  1. Submit. The customer submits a ticket through the support form.
  2. Rule check. The keyword engine runs against the ticket text in real time.
  3. Model call. If no rule matches, one model call returns a category and a confidence score between 0 and 1.
  4. Route. Confidence at or above 0.7 assigns the category automatically. Below 0.7, the ticket is queued for a human.
  5. Record. The final category, whoever assigned it, is stored against the ticket for reporting and for measuring rule and model accuracy over time.

05 · Rollout

Two weeks in shadow, then a 90% accuracy gate to go live

The pipeline never touches real routing until it has proven itself against human classification for two straight weeks.

#PhaseDeliversMeasured by
1Build & tuneWrite the ~50 keyword rules from historical tickets and wire up the model call with the 0.7 confidence threshold.Rule coverage on a sample of past tickets.
2Shadow mode (2 weeks)Rules and model classify every incoming ticket in parallel with normal human triage; predictions are logged, not routed.Daily accuracy against the human-assigned category.
3Live switchRule and model output becomes the real routing decision. Only tickets below 0.7 confidence still reach a human.Accuracy holds at 90%+ after cutover, plus queue volume and per-category error rate.
Verify first
Shadow mode must clear 90% accuracy against human labels before any ticket is auto-routed. If it doesn't, the rules and model prompt get tuned and shadow mode runs again, not skipped.
06 · Open Questions

Three decisions still open before build starts

These don't block writing the keyword rules or the pipeline, but they need an answer before rollout.

  • Model choice. Which model handles the fallback call, and whether it's shared across ticket categories or specialized per category.
  • Rule ownership. Who adds and edits the ~50 keyword rules after launch, and how often they get reviewed against new ticket patterns.
  • Queue size after cutover. What share of the 2,000 daily tickets will actually fall below 0.7 confidence. Shadow mode will produce the real number before launch.
